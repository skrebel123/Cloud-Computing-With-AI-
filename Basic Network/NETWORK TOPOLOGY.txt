NETWORK TOPOLOGY 
=================
WHAT IS TOPOLOGY ?

NETWORK TOPOLOGY IS THE INTERCONNECTED PATTERN OF NETWORK ELEMENTS.
A NETWORK TOPOLOGY MAY BE PHYSICAL, MAPPING HARDWARE CONFIGURATION,
OR LOGICAL, MAPPING THE PATH THAT THE DATA MUST TAKE IN ORDER TO 
TRAVEL AROUND THE NETWORK.

TYPES

1.PHYSICAL TOPOLOGY 

HOW THE PC/LAPTOP/SERVER AND NETWORKING DEVICES ARE PHYSICALLY INTERCONNECTED WITH EACH OTHER 
USING COMMUNICATION MEDIUM.

EXAMPLE
=======
STAR
RING
MESH
BUS


2.LOGICAL TOPOLOGY 

HOW DATA IS FLOWING THROUGH THE PHYSICAL TOPOLOGY IS KNOWN AS LOGICAL TOPOLOGY.

A)TOKEN RING
B)TOKEN BUS


STAR TOPOLOGY
===========================
1.ONE CENTRALIZED DEVICE IS NEEDED.(HUB/SWITCH)
2.NODES ARE CONNECTED WITH SW WITH UTP CABLE (ST.CABLE)+RJ45 JACK 
3.EASY TO CONFIGURE 
4.TROUBLESHOOTING IS EASY 
5.USE OF CABLES ARE LARGE 
6.EACH & EVERY NODE HAS EQUAL ACCESS.


ADVANTAGES OF STAR TOPOLOGY
===========================
.IT IS VERY RELIABLE - IF ONE CABLE OR DEVICE FAILS THEN ALL THE OTHERS WILL STILL WORK.

.IT IS HIGH-PERFORMING AS NO DATA COLLISIONS CAN OCCUR. 

.IT IS LESS EXPENSIVE BECAUSE EACH DEVICE ONLY NEEDS ONE I/O PORT AND WISHES TO BE CONNECTED TO THE HUB WITH ONE LINK.

.EASIER TO PUT IN.

.ROBUST IN NATURE.

.EASY FAULT DETECTION BECAUSE THE LINKS ARE OFTEN EASILY IDENTIFIED.

.NO DISRUPTIONS TO THE NETWORK WHEN CONNECTING OR REMOVING DEVICES.

.EACH DEVICE REQUIRES JUST ONE PORT I.E. TO ATTACH TO THE HUB/SWITCH.

.IF N DEVICES ARE CONNECTED TO EACH OTHER IN STAR, THEN THE AMOUNT OF CABLES REQUIRED TO ATTACH THEM IS N. SO,IT'S EASY TO LINE UP.



DISADVANTAGES OF STAR TOPOLOGY
====================================
.REQUIRES MORE CABLE THAN A LINEAR BUS.

.IF THE CONNECTING NETWORK DEVICE (network switch) FAILS,THE NODES ATTACHED ARE DISABLED AND CAN'T PARTICIPATE IN NETWORK COMMUNICATION.

.MORE EXPENSIVE THAN LINEAR BUS TOPOLOGY DUE TO THE VALUE OF THE CONNECTING DEVICES (network switches).

.IF THE SWITCH GOES DOWN EVERYTHING GOES DOWN, NONE OF THE DEVICES CAN WORK WITHOUT THE HUB.

.SWITCH REQUIRES MORE RESOURCES AND REGULAR MAINTENANCE BECAUSE IT'S THE CENTRAL SYSTEM OF STAR.

.EXTRA HARDWARE IS REQUIRED (SWITCHES) WHICH ADDS TO THE COST. 

.PERFORMANCE IS PREDICATED ON THE ONE CONCENTRATOR I.E. SWITCH.

RING TOPOLOGY
==============
1.EVERY NODES ARE CONNECTED WITH EACH OTHER 
2.EACH AND EVERY NODE HAS EQUAL ACCESS.
3.TOKEN IS USED TO TRANSFER THE DATA 
(TOKEN IS A LOGICAL CONTAINER TO TRANSFER THE DATA IN RING TOPOLOGY)
4.CABLE FAULT ARE EASILY LOCATED
5.IF ONE DEVICE GOES DOWN ENTIRE NW WILL GO DOWN 
6.TO MAKE THIS TOPOLOGY REDUNDANT WE CAN IMPLEMENT DUAL RING TOPOLOGY
A)CDDI-COPPER DISTRIBUTED DATA INTERFACE 
B)FDDI-FIBER DISTRIBUTED DATA INTERFACE


ADVANTAGES OF RING TOPOLOGY
============================
.IN THIS DATA FLOWS IN ONE DIRECTION WHICH REDUCES THE CHANCE OF PACKET COLLISIONS.

.IN THIS TOPOLOGY ADDITIONAL WORKSTATIONS CAN BE ADDED AFTER WITHOUT IMPACTING PERFORMANCE OF THE NETWORK.

.EQUAL ACCESS TO THE RESOURCES.

.THERE IS NO NEED OF SERVER TO CONTROL THE CONNECTIVITY AMONG THE NODES IN THE TOPOLOGY.

.IT IS CHEAP TO INSTALL AND EXPAND.

.MINIMUM COLLISION.

.DUE TO THE PRESENCE OF TOKEN PASSING THE PERFORMANCE OF RING TOPOLOGY BECOMES BETTER THAN BUS TOPOLOGY UNDER HEAVY TRAFFIC.

.EASY TO MANAGE.

.RING NETWORK IS EXTREMELY ORDERLY ORGANIZED WHERE EVERY DEVICE HAS ACCESS TO THE TOKEN AND THEREFORE THE OPPORTUNITY TO TRANSMIT.


DISADVANTAGES OF RING TOPOLOGY
================================
.DUE TO THE UNI-DIRECTIONAL RING, A DATA PACKET (TOKEN) MUST HAVE TO PASS THROUGH ALL THE NODES.

.IF ONE WORKSTATION STUTS DOWN, IT AFFECTS WHOLE NETWORK OR IF A NODE GOES DOWN ENTIRE NETWORK GOES DOWN.

.IT IS SLOWER IN PERFORMANCE AS COMPARED TO THE BUS TOPOLOGY.

.IT IS EXPENSIVE.

.ADDITION AND REMOVAL OF ANY NODE DURING A NETWORK IS DIFFICULT AND MAY CAUSE ISSUE IN NETWORK ACTIVITY.

.DIFFICULT TO TROUBLESHOOT THE RING.

.IN ORDER FOR ALL THE COMPUTER TO COMMUNICATE WITH EACH OTHER, ALL COMPUTER MUST BE TURNED ON.

.TOTAL DEPENDENCE IN ON ONE CABLE.

.THEY WERE NOT SCALABLE.


ADV--
1. DATA FLOWS IN ONE DIRECTION SO THERE IS NO CHANCE OF DATA COLLISION.


DIS ADV--
1.ADDING /REMOVING ANY OTHER DEVICES WILL AFFECT THE WHOLE NW
2.INSECURE AS DATA PASSES THROUGH THE EACH NODE IN A NW
3.TRANSMISSION SPEED IS VERY LOW BCOZ DATA IS MOVING IN ONE DIRECTION AND ONE TOKEN AT A TIME.

MESH
===============
1.EACH AND EVERY DEVICE ARE CONNECTED EACH OTHER 
2.HIGH LEVEL OF REDUNDANCY== DATA MUST REACH TO THE DESTINATION.
3.COST IS VERY HIGH
4.NOT EASY TO TROUBLESHOOT
5.IT TRANSFER DATA TO DIFFERENT NODE SIMULTANEOUSLY.
6.MORE CABLES ARE REQUIRED 
7.COMPLEX NW DESIGN 
8.DIFFICULT TO MAINTAIN
9.MESH TOPOLOGY ARE OFTWO TYPES

A)FULL MESH ====EVERY NODES ARE CONNECTED WITH ALL OTHERS NODES.
B)PARTIAL MESH==>SOME PORTION OF THE NETWORK ARE INTERCONNECTED WITH ALL OTHERS NODES AND FEW NODES ARE CONNECTED WITH SOME NODES BUT NOT ALL.
